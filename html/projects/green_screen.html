Green Screen Algorithm

by Prithviraj Vernekar · Source code · DIY

Have you ever seen people over videos wherein the background of the video changes but the person stays on the same position? You must have heard about green screens for that matter. Ever wondered how that worked?

The Green Screen Algorithm is one the easiest yet very capable of achieving the above task. Normally the algorithm is used for video rendering, but for simplicity we will discuss how it works using images. Speaking of using the algorithm for videos, the algorithm is the same, but it acts on every single frame of the video and renders each individual frame of the video.

As with video rendering, we usually see a person or object infront of a green screen and the video rendering happens at the computer. But so as to work with images, we will first need an image with an object placed over a green coloured background. See Fig. 1.

Once we have an object which will be replaced over a background image of our choice, we will need to look at some constraints before we begin with the algorithm. The algorithm runs in way such that at every iteration (while looping through the pixels of the image), it iterates through the pixels of both the images – the background as well as the image with the object over the green background. So it is required that the dimensions of both the images match. Let the background image be 'bg_image' and the image with the object over the green background be 'foreground_image'.

Once we choose the 'bg_image', we can begin with algorithm.

Let us say we want to save the new image in 'output'. As we begin with the iteration through the 'foreground_image' and 'bg_image', we check the value of the pixel in 'foreground_image', if the value of the current pixel is greater than or equal to certain threshold (say 235), we select the pixel from the 'bg_image' and put it into 'output', (b2 – refer Fig. 2) else we select the pixel from the 'foreground_image' and put it into 'output' (f1 – refer Fig. 2). The visual representation of the process is shown in Fig. 2.

If you consider a group of pixels from 'foreground_image' as well as 'bg_image', the pixel replacement into 'output' is done as shown in Fig. 3.

So if you run the algorithm with Fig. 1 with a background image, you can achieve an appreciable output image with the tiger placed over the background. Though it might seem not good enough to fit into such background, but adjusting the light exposure to the images can produce much better results. This implementation is done just to learn and visualize the working of the algorithm. Well there are much better and efficient ways to do this.

As with video, the studios where such recordings are done, they are basically well lit and mostly have adjusted lights to make it look good. If this is done with better tools and a well lit environment, then the results can be un-noticeable.
